# Blind evaluation split

Frozen artifacts that separate **what the pipeline may learn from** (`dev`) from
**what it is graded on** (`test`). This exists to fix the train/test leakage
documented in [REFACTOR.md](../../docs/REFACTOR.md) §6: the same reviewed
documents were previously used to tune NLU thresholds, seed the LLM few-shot
review index, *and* report precision/recall, which makes any reported number
optimistic by construction.

## Files

| File | Meaning |
|------|---------|
| `dev_split.json` | Doc ids the pipeline MAY use for threshold calibration and few-shot seeding. |
| `test_split.json` | **Blind** doc ids. Reported on exactly once, never used for tuning. |
| `split_manifest.json` | Protocol, holdout fraction, per-family breakdown, `gold_sha256`. |
| `../eval_holdout_ids.json` | The test ids in the shape the tuners read to quarantine them (`eval_common.load_holdout_ids`). |

All four are generated by [`scripts/make_eval_split.py`](../../scripts/make_eval_split.py)
and are deterministic — re-running reproduces them byte-for-byte. Verify with:

```bash
python scripts/make_eval_split.py --check
```

## Protocol

- **Source of truth:** `data/model_technique_map_reviewed.json` (35 frozen,
  human-reviewed documents as of 2026-06-19).
- **Stratification:** by provider family (anthropic, openai, google, meta, …),
  ~30% of each family held out as `test`. Families of size 1 stay wholly in
  `dev` (too small to stratify).
- **Selection:** the first *k* doc ids in sorted order within each family — no
  RNG, no embedded timestamp, so the split is reproducible and auditable.
- **Drift guard:** `gold_sha256` is the SHA-256 of the sorted gold key set. If
  the reviewed corpus changes, the hash changes and the split must be re-frozen
  **and the new test docs re-reviewed before reporting on them.**

## Quarantine (what "blind" enforces)

`test` ids are excluded from:

- `llm_assisted_extraction._build_review_index` — no blind doc seeds a few-shot
  positive/negative example (B.1.2).
- `scripts/calibrate_thresholds.py` — calibration reads `dev` only and refuses
  to load `test` (B.1.3).

Report on `test` **once**, with the final configuration. Repeatedly peeking at
the test set and adjusting in response re-introduces leakage through the
researcher.

## ⚠️ Inter-annotator agreement (κ) — not yet computable

A proper blind gold set is labelled by **≥2 independent annotators** so that
Cohen's/Fleiss' κ can quantify label reliability. This corpus currently has a
**single annotator** (the maintainer), so κ is **undefined**, and the headline
metrics below should be read as *single-rater* numbers.

The capability is built and waiting for data:
[`scripts/annotator_agreement.py`](../../scripts/annotator_agreement.py) computes
κ as soon as a second-annotator file exists at `data/eval/annotations/<annotator>.json`
(same `{doc_id: [techniqueId, …]}` shape). Until then, treat reported precision/
recall as provisional and **do not** claim inter-annotator reliability.
